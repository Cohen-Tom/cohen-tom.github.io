Qu'est-ce qu'une interface en Java ?

En Java, une interface est un contrat qui définit un ensemble de méthodes que les classes peuvent implémenter. Une interface ne contient que des déclarations de méthodes (sans corps) et des constantes. Les classes qui implémentent une interface doivent fournir une implémentation pour toutes les méthodes déclarées dans l'interface.

Les interfaces sont utilisées pour définir des comportements communs que différentes classes peuvent partager, même si elles n'ont pas de relation d'héritage directe. Elles permettent également la programmation orientée objet en permettant l'héritage multiple, ce qui n'est pas possible avec les classes. Par exemple, une classe peut implémenter plusieurs interfaces, ce qui lui permet de partager des comportements avec plusieurs types d'objets différents: 


public interface Volant {
    void voler();
}

public interface Nageant {
    void nager();
}

public class Canard implements Volant, Nageant {
    @Override
    public void voler() {
        System.out.println("Le canard vole.");
    }

    @Override
    public void nager() {
        System.out.println("Le canard nage.");
    }
}

Ici, la classe `Canard` implémente les deux interfaces `Volant` et `Nageant`, ce qui lui permet de disposer des comportements de vol et de nage. Contrairement aux classes, une classe ne peut implémenter qu'une seule classe parente, mais elle peut implémenter plusieurs interfaces.

Pourquoi utiliser des interfaces ?

Les interfaces permettent de définir des comportements communs que différentes classes peuvent partager, même si elles n'ont pas de relation d'héritage directe. Elles facilitent également la programmation orientée objet en permettant l'héritage multiple, ce qui n'est pas possible avec les classes.
